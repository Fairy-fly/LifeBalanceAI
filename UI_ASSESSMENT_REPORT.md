## ❌ 主要缺点和问题

### 问题1：配色系统过于简单

**现状代码分析：**
```cpp
// designtokens.h 中只定义了
inline QString primary() { return color("#5F8E73"); }      // 主绿色
inline QString accent() { return color("#D99467"); }       // 橙色
inline QString danger() { return color("#B75E56"); }       // 红色

// 缺少的颜色
- 没有成功色 (#52C41A)
- 没有进行中色 (#FFB84D)
- 没有休息色 (#A0AEC0)
- 没有维度色 (饮食/运动/睡眠/心理各一个)
- 没有中间色阶 (Light / Lighter / Lightest)
```

**影响：**
- 无法清晰区分不同的健康维度（饮食、运动、睡眠等）
- 缺乏状态色反馈（完成、进行中、待开始）
- UI 显得单调，不够现代

**改进指标：** 颜色从 3 个扩展到 40+ 个

---

### 问题2：缺乏动画和微交互

**现状分析：**
```cpp
// 底部导航栏有基础的指示器动画
// 但缺少的有：
❌ 页面过渡动画
❌ 按钮悬停/按下动画
❌ 卡片交互反馈
❌ 加载动画
❌ 打卡完成反馈动画
❌ 列表项进入动画
```

**QSS 中的动画定义：**
```qss
/* 现状：只有简单的静态样式，没有 transition */
QPushButton:hover {
    background: #F5F5F5;
    border-color: #D9D9D9;
    /* 缺少 transition: all 150ms ease-out; */
}
```

**影响：**
- 用户交互缺乏即时反馈
- 操作不够流畅，显得卡顿
- 缺乏 wow factor（惊喜感）

**改进指标：** 需要添加 20+ 个动画定义
