现在进入 Phase 6：移动端视觉质量二次提升。



背景：

Phase 1-4 已经完成了可用性、页面结构、底部导航、弹窗和回归验收。现在 App 能用，但整体 UI 仍然不够满意，存在 Demo 感、Qt Widgets 控件感、字体偏硬、卡片偏碎、按钮主次不清、部分插画背景不统一等问题。



本阶段目标：

不是继续修功能，而是提升整体视觉品质，让 LifeBalance\_AI 更像正式、温和、干净、有品质感的移动端健康陪伴 App。



当前不要处理：



\* `LifeBalanceAI\_demo/tools/`

\* 文档归档

\* tag

\* README

\* 构建产物



不要自动提交，不要暂存，不要 `git add .`。



开始前先执行：



1\. `git status --short`

2\. `git log --oneline -8`

3\. 确认当前只剩未跟踪项 `LifeBalanceAI\_demo/tools/`

4\. 不处理 tools 目录。



注意：

用户提到的“下移 5cm / 降低 3cm / 间距 2cm”等，是基于真机截图的视觉描述，不要机械按物理厘米写死。请根据 393dp 左右宽度的 Android 真机截图换算成合理的 dp / px，并用截图反复校准。原则是“视觉上明显达到目标，但不破坏小屏适配”。



\## Phase 6 总体视觉方向



保留现有方向：



\* 奶油背景；

\* 温和绿色主色；

\* 健康陪伴感；

\* 低饱和；

\* 圆角卡片；

\* 不做复杂渐变；

\* 不做重阴影；

重点修正：



\* 字体太硬、太粗；

\* 弹窗样式不完全统一；

\* Profile 用户画像卡片底部留白过多；

\* 部分插画背景与页面背景不一致。

\# Phase 6A：视觉系统二次打磨



本小阶段先处理全局视觉系统，不大改页面结构。



\## 目标



统一字体、按钮、卡片、弹窗、图表基础样式，让后续页面精修有稳定基线。



\## 重点任务



1\. 全局字体权重降噪



\* 页面大标题可以保持粗体；

\* 卡片标题改为 medium / semi-bold；

\* 正文不要大面积 bold；

\* 辅助文字颜色更浅；

\* AI 报告、周报详情、长期目标正文尤其要减少粗体压迫感。



2\. “LifeBalance AI”标题字体优化



\* 登录页的 `LifeBalance AI` 当前字体不好看，过硬。

\* 换成更圆润但不细的字体风格。

\* 优先使用系统可用字体，不要引入版权不明字体。

\* 可尝试字体优先级：



&#x20; \* MiSans / HarmonyOS Sans / Noto Sans CJK SC / Noto Sans

&#x20; \* 英文标题可尝试更圆润的 bold / demi-bold 风格

\* 如果没有合适字体，不要强行引入外部字体，使用 Qt 字体加粗、字间距、圆润 fallback 做近似。



3\. 按钮层级统一



\* 主按钮：绿色实心；

\* 次按钮：浅底 / 描边；

\* 危险按钮：柔和红，不要高饱和刺眼红；

\* 禁用按钮：低对比浅灰；

\* 首页“反馈 / 调整”必须明确主次：



&#x20; \* 未完成计划：反馈为主操作，调整为次操作；

&#x20; \* 已反馈计划：修改反馈可以作为次主操作，调整应更弱；

&#x20; \* 不再让橙色和蓝色两个按钮同时抢视觉权重。



4\. 弹窗系统统一

&#x20;  统一以下弹窗的视觉规则：



\* 反馈弹窗；

\* 调整规划弹窗；

\* 昨日回顾；

\* AI 分析报告；

\* 周报提示；

\* 周报详情；

\* 导出成功；

\* 退出确认。



统一要求：



\* 柔和遮罩；

\* 面板居中或底部安全区域内稳定；

\* 圆角统一；

\* 边框减弱；

\* 标题、正文、按钮区层级清楚；

\* 按钮不被底部导航遮挡；

\* 不使用旧的左侧竖条提示样式；

\* 输入框不要出现突兀的蓝色水滴 / 系统拖拽柄影响视觉。



5\. 周报图表字体 / 轴标签修复



\* 周报折线图 Y 轴中文标签现在有乱码或异常风险。

\* 优先移除中文 Y 轴标题。

\* 图表只保留轻量网格、折线、日期和必要数字。

\* 不要在小尺寸移动端图表里使用竖排中文轴标题。

\* 如果必须显示轴标签，使用 0 / 50 / 100 等纯数字，避免 Android 字体渲染问题。



允许修改：



\* `designtokens.h`

\* `resources/style.qss`

\* `uifactory.cpp`

\* `animateddialog.cpp`

\* `mainwindow.cpp`

\* 图表相关绘制代码



禁止修改：



\* 数据库；

\* Auth；

\* AI 请求；

\* Prompt；

\* 业务逻辑；

\* schema；

\* 构建产物；

\* 未跟踪 tools。

\# Phase 6B：登录页 / 注册页 / 过渡页精修



这是第一眼观感阶段，优先级很高。



\## 1. 登录页



当前用户要求：



\* logo 图片太小，需要放大约 1.5 倍；

\* 登录页整体向下调整，视觉上大约下移 5cm；

\* `LifeBalance AI` 大标题字体不好看，需要换成有圆角、但不细的字体。



执行要求：



\* logo 放大 1.4-1.6 倍，最终以真机截图为准；

\* 页面整体下移，但不能导致底部“立即注册”被手势条或底部区域挤压；

\* 标题字体更圆润、更友好，但仍有力量感；

\* 标题不要像默认 italic 字体或廉价 logo 字体；

\* 登录卡片内部间距保持稳定；

\* 输入框高度、按钮高度不要继续变大；

\* 键盘弹出时仍可操作。



注意：

不要只靠增加顶部 spacer 粗暴下推，必须检查 390x844、393x852、414x896 这类尺寸是否还能完整显示核心操作。



\## 2. 注册页



当前用户要求：



\* 注册页控件几乎都在上面，显得头重脚轻；

\* “开始你的健康陪伴”标题放大约 1.2 倍；

\* 标题和下面控件距离加大，视觉上大约多隔开 2cm；

\* 去掉“选择角色”，因为新用户不能直接选择律行者这类高级用户。



执行要求：



\* 标题放大 1.15-1.25 倍；

\* 标题下方增加更舒适的留白；

\* 去掉角色选择 UI；

\* 注册逻辑中为新用户设置默认初始角色，例如“探索者”或系统默认普通角色；

\* 不允许用户注册时直接选择“律行者 / 高级用户”；

\* 保持手机号、密码、验证码、发送验证码、注册按钮布局稳定；

\* 页面不要头重脚轻，也不要底部空得过分。



注意：

如果删除角色选择会影响注册参数，请只做最小业务适配：传默认角色，不改账号体系。



\## 3. 过渡页 / 启动过渡图



当前问题：



\* 过渡图文案“今天也要好好照顾自己呀”占屏幕宽度太大；

\* 视觉上被长文字牵着走；

\* 需要重新评审并生成新的过渡图方案。



执行要求：



\* 不建议把中文长文案直接嵌进图片。

\* 更推荐生成“无文字插画背景”，然后在 Qt 里用 QLabel 单独渲染短文案。

\* 过渡页文案建议更短，例如：



&#x20; \* “慢慢来，也很好”

&#x20; \* “今天，从一点点开始”

&#x20; \* “照顾自己，从现在开始”

&#x20; \* “让健康轻一点”

\* 插画风格保持：



&#x20; \* 奶油背景；

&#x20; \* 温和绿色；

&#x20; \* 健康陪伴；

&#x20; \* 小机器人 / 叶子 / 餐食 / 运动 / 记录元素；

&#x20; \* 不要大面积文字；

&#x20; \* 不要高饱和；

&#x20; \* 不要复杂细节。



如果可以调用 images2：



\* 重新生成过渡页插画；

\* 生成时优先透明背景或与页面奶油背景完全一致；

\* 不要把长中文放进图片；

\* 输出 1 张竖屏过渡图和 1 张横向/顶部装饰图备选。



如果不能调用 images2：



\* 不要假装生成图片；

\* 输出可直接用于 images2 的图像生成提示词；

\* 说明需要替换的资源路径；

\# Phase 6C：首页与高频操作精修



\## 1. 首页餐食卡片按钮



当前问题：



\* “反馈 / 调整”两个按钮同时抢视觉；

\* 颜色主次不清；

\* 已完成状态下“修改反馈 / 调整”层级仍混乱。



整改要求：



\* 反馈是主操作；

\* 调整是次操作；

\* 已反馈后，修改反馈显示为较轻的主动作；

\* 如果某餐已经完成，调整可以降级、置灰或弱化；

\* 不要继续使用橙色/蓝色并列抢视觉。



\## 2. 长期目标



当前问题：



\* 长期目标文字密度高；

\* 展开后阅读压力大；

\* 默认状态不够摘要化。



整改要求：



\* 默认折叠时显示摘要，而不是大段全文；

\* 尝试提取 2-3 个目标 chips：



&#x20; \* 每周 4 次训练；

&#x20; \* 蛋白质 160g+；

&#x20; \* 体重目标 79kg；

\* 展开后再显示全文；

\* chips 低饱和、轻边框，不要变成彩色标签堆。



\## 3. 反馈弹窗



当前问题：



\* 输入框旁边出现突兀蓝色水滴 / 系统视觉；

\* “选择图片 / 预览”按钮不够整齐；

\* 弹窗不像统一高质量 modal。



整改要求：



\* 优化输入框 focus 状态，避免突兀蓝色水滴影响截图观感；

\* “选择图片”和“预览”要么等宽，要么合并为一个“添加图片”入口；

\* 预览不可用时明确 disabled；

\* 底部按钮统一两列；

\* 不改图片上传业务逻辑。



\## 4. 调整规划弹窗



当前问题：



\* placeholder 过长，像已有输入；

\* 示例文案过度引导。



整改要求：



\* placeholder 改为“输入你的调整需求...”

\* 示例放在输入框外的浅灰辅助文案；

\* 标题、正文、按钮按统一 modal 规范。
